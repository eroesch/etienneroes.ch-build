Right. Disclaimer: A setup like this one is not for everyone. If you found these lines, you probably already know that. Just to be clear, this setup could not be any further from Wordpress, in a good way.

The team behind Nikola has done an amazing job. My experience so far as been flawless; the documentation, the code, the open repos, the comments in the code have been extremely useful. Everything in my research points to the need to get into what you're doing: Reading a book on a laptop feels less embedded, less efficient, and you may remember less; the embodiment makes everything feels more real. I do prefer pen&paper to keep notes--and I take notes about everything, and yes even write on books--but I do need to get more out of my study time, and I need more study time.

With this open notebook, I achieve quite a few things.

  1. I code as I am studying;
  2. I centralise (and version control) this code;
  3. I tend to comment it out a bit more, because it is in the open;
  4. I share with others, and will I am sure enjoy receiving feedback;
  5. I may even get more and better exposure;

import matplotlib.pyplot as plt
import numpy as np


t1 = np.arange(0.0, 2.0, 0.1)
t2 = np.arange(0.0, 2.0, 0.01)

# note that plot returns a list of lines.  The "l1, = plot" usage
# extracts the first element of the list into l1 using tuple
# unpacking.  So l1 is a Line2D instance, not a sequence of lines
l1, = plt.plot(t2, np.exp(-t2))
l2, l3 = plt.plot(t2, np.sin(2 * np.pi * t2), '--go', t1, np.log(1 + t1), '.')
l4, = plt.plot(t2, np.exp(-t2) * np.sin(2 * np.pi * t2), 'rs-.')

plt.xlabel('time')
plt.ylabel('volts')
plt.title('Damped oscillation')

plt.show()
